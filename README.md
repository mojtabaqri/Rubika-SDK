# Vamban Loan Marketplace

این پروژه یک پلتفرم مدیریت وام (P2P Lending Marketplace) با PHP و روبیکا است.

## ویژگی‌های اصلی

- ✅ ربات روبیکا برای ثبت نام، KYC و ثبت آگهی
- ✅ پنل مدیریت تحت وب با Bootstrap
- ✅ پایگاه داده SQLite برای ذخیره کاربران، KYC، آگهی‌ها، پرداخت‌ها، escrow و گزارش‌ها
- ✅ سیستم اعتبار کاربر (Trust Score)
- ✅ جریان کامل آگهی‌ها: ثبت → تایید مدیر → انتشار
- ✅ سیستم پرداخت دستی و Escrow
- ✅ نمایش آگهی‌های تایید شده در صفحه‌ی اصلی

## ساختار پروژه

```text
app/
├── Database.php         # اتصال SQLite و میگریشن
└── Models.php           # مدل‌ها و helperهای CRUD

config.php               # بارگذاری محیط و تنظیمات
index.php                # صفحه‌ی اصلی نمایش آگهی‌ها
admin.php                # پنل مدیریت
webhook.php              # ورودی webhook روبیکا
ad.php                   # جزئیات آگهی
.env                     # تنظیمات محیطی (غیراجباری، در .gitignore قرار دارد)
```

## نصب و راه‌اندازی

1. پروژه را در پوشه‌ی وب‌سرور خود قرار دهید.
2. در صورت نیاز `composer install` را اجرا کنید.
3. مقدارهای `BOT_TOKEN`, `CHAT_ID`, `ADMIN_USER`, `ADMIN_PASS` و `APP_TITLE` را در فایل `.env` تنظیم کنید.
4. برای بررسی اتصال ربات به این آدرس بروید: `bot_setup.php?action=status`
5. برای ثبت وب‌هوک به این آدرس بروید: `bot_setup.php?action=webhook`
6. وارد `admin.php` شوید تا کاربران، آگهی‌ها و پرداخت‌ها را مدیریت کنید.

## فایل‌های مهم

- `config.php` - بارگذاری تنظیمات محیط و راه‌اندازی پایگاه داده
- `app/Database.php` - اتصال SQLite و ایجاد جداول
- `app/Models.php` - مدل‌های کاربران، KYC، آگهی، پرداخت، escrow و گزارش
- `webhook.php` - پردازش پیام‌های ورودی روبیکا و ثبت KYC/آگهی
- `admin.php` - پنل مدیریت برای تایید کاربران، آگهی‌ها و پرداخت‌ها
- `index.php` - صفحه نمایش آگهی‌های تایید شده
- `ad.php` - صفحه جزئیات آگهی

## نحوه استفاده

- کاربران با ارسال پیام به ربات روبیکا وارد سیستم می‌شوند.
- ربات مرحله‌ای شماره، نام، کد ملی، تاریخ تولد و KYC را جمع‌آوری می‌کند.
- پس از ثبت KYC، مدیر از طریق `admin.php` می‌تواند آن را تایید یا رد کند.
- کاربران تایید شده می‌توانند آگهی درخواست یا ارائه وام ثبت کنند.
- آگهی‌ها پس از تایید مدیر در صفحه اصلی منتشر می‌شوند.

## نکات امنیتی

- حتماً `ADMIN_PASS` را در `.env` تغییر دهید.
- فایل `.env` و دیتابیس SQLite در `.gitignore` قرار دارند.
- پرداخت‌ها باید توسط مدیر به صورت دستی بررسی شوند.

## توسعه‌ی بعدی

برای کامل کردن سیستم می‌توانید:

- اضافه کردن ارسال عکس واقعی و ذخیره فایل‌ها
- کامل کردن سیستم Escrow و ایجاد معاملات واقعی
- افزودن سیستم پیام داخلی بین کاربران
- توسعه UI وب با فیلتر و جستجوی پیشرفته
- ساخت داشبورد آمار دقیق‌تر
